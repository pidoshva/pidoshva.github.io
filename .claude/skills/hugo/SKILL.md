# Hugo Static Site Output

This repo contains **compiled Hugo output**, not Hugo source. No templates,
`config.toml`, or `content/` markdown. All HTML deploys directly to GitHub
Pages from `main`. Edit HTML files in place for content changes.

## Page Types at a Glance

| Page Type | Path | Header ID | Extra Scripts |
|-----------|------|-----------|---------------|
| Homepage | `/index.html` | `#header` | none |
| Post | `/posts/[slug]/index.html` | `#header-post` | `code-copy.js` |
| Posts Archive | `/posts/index.html` | `#header` | none |
| Project | `/projects/[slug]/index.html` | `#header-post` | none |
| Projects List | `/projects/index.html` | `#header` | none |
| Tag Archive | `/tags/[tag]/index.html` | `#header` | none |
| About | `/about/index.html` | `#header-post` | none |

## Common Layout Skeleton

Every page shares this body structure (scripts at end of `<body>`):

```html
<body class="max-width mx-auto px3 ltr">
  <div class="content index py4">
    <header id="header"><!-- or id="header-post" for article pages -->
      <!-- Logo + nav: Home, Projects, Resume (external https://geleus.io/), About -->
    </header>
    <!-- Page-specific content here -->
    <footer id="footer">
      <div class="footer-left">Copyright &copy; 2025 &copy; geleus</div>
      <div class="footer-right"><nav><!-- same 4-link nav --></nav></div>
    </footer>
  </div>
  <link rel="stylesheet" href=/lib/font-awesome/css/all.min.css>
  <script src=/lib/jquery/jquery.min.js></script>
  <script src=/js/main.js></script>
  <!-- Post pages only: <script src=/js/code-copy.js></script> -->
</body>
```

## Workflow: Add a New Blog Post

1. Copy an existing post directory (e.g., `/posts/update-feb-2025/index.html`) to `/posts/new-slug/index.html`
2. Edit the new file: update `<title>`, `og:title`, `og:url`, `canonical`, `article:published_time`, `article:modified_time`, and `<h1 class="posttitle">`
3. Update prev/next links: set the `href` on `.fa-chevron-left` and `.fa-chevron-right` anchors in `#header-post`, and update the **previous** post's next-link to point to this new post
4. Update share URLs: percent-encode the new slug in all 8 sharing `<a>` tags inside `#share` and `#share-footer`
5. Add a `<li class="post-item">` entry in `/posts/index.html` under the correct `<h2>` year heading
6. Add a matching `<li class="post-item">` entry in `/index.html` section `#writing`
7. Add an `<item>` entry in `/posts/index.xml` and `/index.xml` RSS feeds
8. Add a `<url>` entry in `/sitemap.xml` with the current date as `<lastmod>`
9. Verify: `grep -r 'href="/posts/new-slug/"' --include="*.html"` confirms links resolve

## Workflow: Add a New Project

1. Create `/projects/new_project/index.html` using `/projects/this_website/index.html` as template
2. Edit metadata: `<title>`, `og:title`, `og:url`, `canonical`, timestamps
3. Add a `<li>` entry in `/projects/index.html` with `<div class="hn-tag-list">` for technology tags and `<div class="hn-summary">` for the description
4. Add an `<item>` in `/projects/index.xml` and `/index.xml`
5. Add a `<url>` in `/sitemap.xml`

## CSS Hash Warning

One compiled CSS file: `/css/styles.[64-char-hex].css` with SRI integrity.
Every HTML file references it via `<link>`. Changing CSS requires:

1. Rename the CSS file with the new hash
2. Use `Grep` for `styles\.` across all `*.html` files
3. Update every `href` and `integrity` attribute to match

## Key Conventions

| Convention | Value |
|-----------|-------|
| New post slugs | `kebab-case` |
| Project slugs | `snake_case` (legacy convention) |
| Date display | `YYYY-MM-DD` |
| Date meta | ISO 8601 with timezone offset |
| Schema.org | `BlogPosting` on all article pages |
| Internal links | Root-relative (`/posts/...`), never absolute domain URLs |
| External resume | `https://geleus.io/` (only external nav link) |
